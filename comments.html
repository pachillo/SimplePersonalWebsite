Technical Aspects

These website pages are developed according to the UTS Web Assignment specification.
As required, a folder called websystems is created with four HTML files (index.html, past.html, future.html and comments.html) and one CSS file (websystems.css).

My HTML general structure is:

head
body
footer

Each part is completed by the use of div (class or ID).

The head contain all the information that only the browser recieves. These things are the title of the websites, the favicon, others meta tag and link to the CSS stylesheets and viewport.

The body is where almost all of the CSS class and ID are used.

.navBar
#header
main
#gallery (index.html only)

The .navBar is fixed position with a z index of 999 to always stay on top where ever you are on the website. The .navButtons are linked to other pages. The CSS3 ":hover" and .activeButton are responsible for how the button behave. If hovered over the button will change it's display. If the button is on the active page, then it's display will be different from the rest.

The #header cannot be a class because each page has a unique header. But basically each one have a background image and a short line to tell the users about the website. Each page has a unique #header and #title. The background image is done throught CSS by backgroung-image and setting the image width to 100% and height varies.

The main is where most of the class and ID are from. Overall, main is set to have a consistent margin, padding, line height, text size, text color and background color. In index.html, the two text passages can pop up and change color. Once again, this is done by :hover. In other pages, tables and lists can be found. Things are pretty standard here. The past table cells on an even row are colored by ":nth-child(even)". The past has unordered lists within the table, and the future has an ordered list. Images are also in past and future too. In this comment page, hyperlink text is used.

The photo gallery native to index.html is a grid display of 4 cells. A grid is built by declaring the position. Then the image is assign to each postion and cover fit. The biggest image is assigned to 2 cells, while others are assigned to the rest. The ":hover" is used again to show the picture's description if hovered over.

The @media query is used in CSS to create a viewport for mobile devices. Two resolution is considered: width of 768px (tablet or old school monitor) and 576px (smartphone). "@media only screen and (max-width:576px)" this line mean only if that device has a screen with a size of 576 or lower, then the changes will be applied. Same goes with 768px. The NavBar is change in both cases, since the original one would go out of frame, a NavBar with 2 rows is used instead. For 576px, the index's greetings line has to be hightlighted with a transparent black to distinguish it from the background. Other than that, the padding and images are tuned to fit better with these devices. I have to remove the hovering features of text passages and images for both of the smaller screen size, since I don't think it worths much and it affects other element too much.

The footer has a class ".Top" to style to Go Top button. With ":hover", the Go Top button is more interactive. In case you haven't guessed it, it takes you back to the top of the page. However, you won't see this button when you are using a phone. Other than that, the footer also display other basic information.

Aesthetics

My main idea when designing the website is first to captivate the users' attention and then lead them to the rest of the website. All of the design aspects should ultimately support the website's user-friendly elements.

1. Overall

The first thing that comes to my mind is to relate each website to a type of landscape. By doing this, I aim to connect the users somehow to my story since it does not share common feats with everyone. Also,it would make the users feel like they are on a journey. To support my nature-scheme website, the primary colour of the website is chosen to be a shade of dark green. It's not too bright to damage one's eyes but also not too dark to be neglected. As a follow-up, the page's background colour should be white with a strong touch of beige to resemble a book's page. Finally, the colour yellow is added to finish the colour palette. Yellow is a controversial colour, and I know many people who despise it. However, I think some types of yellow go great with green and are pleasing to look at. Movie subtitles inspire these shades of yellow. In the end, my three primary colours are green, yellow and light beige.

The overall layout of these pages are:

Elements (in order) What's in it
NavBar Assignment name and NavButtons
Intro Background image and short intro
Website's content The rest of the website content
Footer Copyright, Uni and subject name, validation

I believe this is a well-structured and modern design for websites, as it is user-friendly and beautiful. The navbar and footer will be the same throughout the pages to maintain a sense of connection. I also added a star favicon because why not.

2. Home Page

This is where the user first lands on when visiting my website. Therefore, it must make a statement and be attractive. To do this, a huge background photo of a mountain with trees and water features was chosen. Next is a huge intro line with a funny little line below to ease the user. Combining these, we have a beautiful opening that resembles a nature documentary/ movie scene. The background photo of many features resembled my present as it combines what was is used later (water and mountain).
Moreover, it will make it seem like the user has just travelled somewhere new (the website's theme). With the greeting intro, the user is treated as a traveller into my "digital realm", creating a personal feeling. Also, the intro line would blend too much with the white background when designing the smartphone view. This was an excellent opportunity to utilise a slightly transparent black background. Once again, it is just like the movie's subtitles.
Next are the two text paragraphs. Each one has a small heading (h2). The headings are short for a minimalistic look. And both will scale up if hovered over, really making them pop. This helps the user focus on what is being written there. The "Me" part is pink, creating a sense of calm and friendliness as I talk about myself. The "My website" part has a blue background, creating trust and relaxation as it gets technical.
Finally, "My Gallery" has three images of what I like. These images are arranged tightly yet seem organised and pleasing to view. This makes my website cleaner and more modern. Hovering over them will give you the description, which help with the minimalistic vibe that I follow.

3. Past, Future and Comment Page

Now that the user is more acclimated to the website, the last three websites will be more straightforward. All will have a smaller background picture, with a small intro to tell them the website's purpose. This makes the users feel more used to the website.
The past website has a beach (a water feature), often used as the lowest point for measuring (sea level). This resembles the past, which is further back in the timeline. The future has a mountain background, which is high and looks open. This resembles the future, which is further ahead in the timeline, and it is pretty uncertain. The comment's background is a desk, which I would also count as a landscape :). This resonates much with the comment page's content, which is a deep dive into how the website is built. In addition, the comment breaks off from the overall theme in some way (the background, the non-movie font colour). The users must stay locked in and prepare for the technical stuff.
All three websites used simple paragraphs with headings and pictures in between to define them better. The past has a table that summarises the entry page. It looks neat and easy to understand, with a nice touch of cell colouring. The future has an ordered list, which is straight to the point. The comment has a combination of both.

Aesthetics are my favourite part of making the website. So, hopefully, you would understand why I talked so much here. If you are wondering how I find such specific colours, I used Windows PowerToys.

Accessibility

While designing the website, accessibility is also taken into account.

  • The navbar is consistent and stays on top of every page for easy navigation.
  • The navbar indicates what page you are on by highlighting the button for that page.
  • The footer is consistent throughout every page to display extra information.
  • The "Go Top" button in the footer let the user easily go back on top of the page.
  • The background color and the text color are highly contrasted with each other, making it easy to read.
  • The chosen font (Arial, Sans serif) is web-safe and helps avoid website crashes or browser incompatibility.
  • The text font and color are very readable.
  • The text passage on the home page pops up, which can help people with low attention span stay focused.
  • Each image has a description and 'alt' should the image not respond.
  • Tables and lists allow a clear view and help the users understand quickly.
  • Design for smartphones and tablets is also considered, with different fitting styles.
